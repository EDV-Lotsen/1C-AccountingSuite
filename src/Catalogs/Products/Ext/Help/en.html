The Items list stores information about items (products or services) that you buy or sell.   

To add an item

  1. In the Lists section, click Items and then click Create.
  2. Enter or select the following:
    • Code. By default the system assigns consecutive codes when items are saved, and no user input is required.

      You have the option to enter a custom code instead. Codes can include prefixes. For example, if you specify the code "I0001", your next purchase invoices are automatically numbered "I0002", "I0003", and so on.  

      By default the system will assign 15-digit item codes. If you want to use shorter codes, enter a shorter code when creating the first item (for example, 00001), and then the system will automatically assign shorter consecutive codes (00002, 00003, and so on).
       
    • Description. The item name as it appears in lists and documents.
    • Type. Select one of the following:
      • Inventory. The system will track inventory balances for this item, and calculate cost of goods sold based on the selected costing method.
      • Non-Inventory. The system will not track inventory balances for this item. Use this option for services, or for items that do not require inventory balance tracking or cost of goods sold calculation.
    • Unit of measure. The unit of measure for the item.

      The selected unit of measure will appear in document print forms.
       
    • Costing method. The costing method.

      It is only applicable to inventory items.

      If the Disable LIFO (IFRS) check box is selected in Accounting settings, you cannot save an item with LIFO costing method.
       

    • Inventory / expense account, Income account, and COGS reclass account. The accounts used for automatic G/L posting in the General Journal.

      AccountingSuite performs automatic reclassification of inventory cost to COGS for sales transactions of inventory items. It does not perform reclassification for noninventory items, therefore you cannot select a COGS reclass account for noninventory items.

  3. If the VAT check box is selected in Accounting settings, the Purchase VAT code and Sales VAT code fields will be prefilled with default values.
  4. Click Write and close.

    The informational (read-only) section at the bottom of the item form contains the following data:
    • Item cost.
      • Last cost. The cost taken from the last purchase invoice.
      • Average cost. The weighted average cost.
      • Accounting cost.  The cost calculated according to the selected costing method.
    • Item quantities.
      • Qty on PO. Items on purchase orders (ordered but not yet received).
      • Qty on SO. Items on sales orders (ordered but not yet shipped).
      • On hand. Items available.
      • Available to promise. Quantity on hand + items on purchase orders - items on sales orders.
