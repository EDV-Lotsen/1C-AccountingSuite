The purchase return document records an item return to a vendor, and also can be used for entering beginning balances.

The sales return workflow supports the following scenarios: payments from vendor for returned goods and issuing a credit towards your next purchases.

Purchase returns update the inventory balances and generate general journal transactions. For detailed information, see the Inventory journal and General journal help sections.

You have the option to create a purchase return document based on a Purchase invoice document. In this case most of the purchase return fields are automatically filled from the purchase invoice. To ensure that the purchase return has the most accurate cost information, it is recommended that you create it from a purchase invoice.

To enter a purchase return

  1. In the Purchasing section, click Purchase returns, and then click Create.
  2. Enter or select the following:
    • Number. By default the system assigns consecutive numbers when documents are saved, and no user input is required.

      You have the option to enter a custom number instead. Document numbers can include prefixes. For example, if you specify the number "PR-1", your next purchase returns are automatically numbered "PR-2", "PR-3", and so on. 
    • Currency. The transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • A/P account. The accounts payable account that is used for G/L posting.
    • Price includes VAT. Indicates whether the prices specified in the purchase return include VAT.

      This field is only available if the VAT check box is selected in Accounting settings. 
    • Date. The purchase return date.

      This field is prefilled with the current date.
    • Exchange rate (shown as currency symbols, for example, $/1€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the purchase return date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate prices in the line items, therefore it is recommended that you specify the date and currency before entering the line items.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • RMA/Ref#. The number of the vendor document related to the purchase return, such as a Return Merchandise Authorization.
    • Return type. Select one of the following:
      • Refund. The payment is made by the vendor.
      • Credit memo. The amount is used as a credit towards future purchases.
    • Vendor. The vendor (here you can only select companies of the vendor type).

      The field name can be customized (for example, to Supplier) in the Vendor name field in Accounting settings. 
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings.
    • Location. The location of the warehouse from which the purchase is returned.

      This field is only available if the Multilocation check box is selected in Accounting settings. 
    • Purchase invoice. The purchase invoice that the purchase return is based on.
  3. In the Line items list, select the items included in the purchase return, and specify their prices and quantities.

    Item prices are automatically prefilled with item last costs. The system automatically calculates a line total for each item.

    If the VAT check box is selected in Accounting settings, a default VAT code is prefilled for each item.
  4. Enter or select the following:
    • Memo. In this field you can enter notes on the purchase return.
    • Beginning balance. Indicates whether the purchase return is used for entering beginning balances.

      This field is only available if the Beg. balances in documents check box is selected in Accounting settings.
  5. Click Post and close.

    The informational (read-only) section at the bottom of the form contains the following data:
    • VAT. VAT calculated based on the VAT codes assigned to items.

      When you post the purchase return, a VAT liability is recorded to the General journal.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Total FCY. The total amount in the transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings. 
    • Total. The total amount in the reporting currency.
