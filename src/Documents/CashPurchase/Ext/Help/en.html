The cash purchase document incorporates a purchase invoice and an invoice payment purchasing steps into a single transaction. It is essentially an ad hoc purchasing document.

Cash purchases generate general journal transactions. For detailed information about transactions that are generated when a cash purchase is posted, see the General journal help section.

To enter a cash purchase

  1. In the Purchasing section, click Cash purchases, and then click Create.
  2. Enter or select the following:
    • Number. By default the system assigns consecutive numbers when documents are saved, and no user input is required.

      You have the option to enter a custom number instead. Document numbers can include prefixes. For example, if you specify the number "CP-1", your next cash purchases are automatically numbered "CP-2", "CP-3", and so on. 
    • Currency. The transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • Bank account. The account (e.g. bank, cash, credit card, PayPal, etc.) used for the transaction.
    • Price includes VAT. Indicates whether the prices specified in the cash purchase include VAT.

      This field is only available if the VAT check box is selected in Accounting settings. 
    • Date. The cash purchase date.

      This field is prefilled with the current date.
    • Exchange rate (shown as currency symbols, for example, $/€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the cash purchase date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate prices in the line items, therefore it is recommended that you specify the date and currency before entering the line items.

      This field is only available if the Multicurrency check box is selected in Accounting settings.  
    • Vendor. The vendor (here you can only select companies of the vendor type).

      The field name can be customized (for example, to Supplier) in the Vendor name field in Accounting settings. 
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings.
    • Location. The location of the warehouse where the purchase is delivered.

      This field is only available if the Multilocation check box is selected in Accounting settings. 
  3. On the Line items tab, select the items included in the cash purchase, and specify their prices and quantities.

    Item prices are automatically prefilled with item last costs. The system automatically calculates a line total for each item. 

    If the VAT check box is selected in Accounting settings, a default VAT code is prefilled for each item. 
  4. In the Memo field, you can enter notes on the cash purchase.
  5. Click Post and close.

    The informational (read-only) section at the bottom of the form contains the following data:
    • VAT. VAT calculated based on the VAT codes assigned to items.

      When you post the cash purchase, a VAT liability is recorded to the General journal.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Total FCY. The total amount in the transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings. 
    • Total. The total amount in the reporting currency.
