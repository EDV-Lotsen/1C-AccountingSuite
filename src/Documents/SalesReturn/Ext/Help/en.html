The sales return document records an item return by a customer, and also can be used for entering beginning balances. 

The sales return workflow supports the following scenarios: payments to customer for returned items and issuing a credit towards their next purchases.

Sales returns update the inventory balances and generate general journal transactions. For detailed information, see the Inventory journal and General journal help sections.

You have the option to generate a sales return document from a Sales invoice document. In this case most of the sales return fields are automatically filled from the sales invoice. To ensure that the sales return has the most accurate cost information, it is recommended that you create it from a sales invoice.

To enter a sales return

  1. In the Sales section, click Sales returns, and then click Create.
  2. Enter or select the following:
    • Number. By default the system assigns consecutive numbers when documents are saved, and no user input is required.

      You have the option to enter a custom number instead. Document numbers can include prefixes. For example, if you specify the number "SR-1", your next sales returns are automatically numbered "SR-2", "SR-3", and so on. 
    • Return type. Select one of the following:
      • Refund. The payment is made to the customer.
      • Credit memo. The amount is used as a credit towards future sales.
    • Currency. The transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • Price includes VAT. Indicates whether the prices specified in the purchase return include VAT.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Date. The sales return date.

      This field is prefilled with the current date.
    • A/R account. The accounts receivable used for G/L posting.
    • Exchange rate (shown as currency symbols, for example, $/1€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the sales return date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate prices in the line items, therefore it is recommended that you specify the date and currency before entering the line items.
    • Ref#. Optional transaction number (such an RMA, return merchandise authorization, number given to the customer).
    • Customer. The customer (here you can only select companies of the customer type).

      The field name can be customized (for example, to Client) in the Customer name field in Accounting settings. 
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings.
    • Location. The location of the warehouse to which the sale is returned.

      This field is only available if the Multilocation check box is selected in Accounting settings. 
    • Sales invoice. The sales invoice that the sales return is based on (read-only).
  3. In the Line items list, select the items included in the sales return, and specify their prices and quantities.

    Item prices are automatically prefilled from the Price list. The system automatically calculates a line total for each item.

    If the VAT check box is selected in Accounting settings, a default VAT code is prefilled for each item. 

    If the Sales tax (US) check box is selected in Accounting settings, it also automatically calculates the sales tax based on the buyer's tax code selected in the Company window.  
  4. Enter or select the following:
    • Memo. In this field you can enter notes on the sales return.
    • Beginning balance. Indicates whether the sales return is used for entering beginning balances.

      This field is only available if the Beginning balances check box is selected in Accounting settings.
  5. Click Post and close.

    The informational (read-only) section at the bottom of the form contains the following data:
    • Sales tax. The sales tax based on the sales tax code assigned to the customer company and the sales tax types assigned to line items.
    • VAT. VAT calculated based on the VAT codes assigned to items.

      When you post the sales invoice, a VAT liability is recorded to the General journal.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Total FCY. The total amount in the transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings. 
    • Total. The total amount in the reporting currency.
