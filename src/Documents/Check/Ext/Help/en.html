The check is a document used to record payment transactions. There are several business processes where checks can be used:
  • Checks are created based on Invoice payment or Cash purchase documents to automate physical check writing.
  • Checks are created manually to create transactions that record payments to vendors, and optionally to automate physical check writing.
  • Checks are created manually to create miscellaneous expense transactions without assigning them to any vendor.
     

Checks generate general journal transactions. For detailed information about transactions that are generated when a check is posted, see the General journal help section.

If you generate a check based on an Invoice payment or a Cash purchase, most of the check fields are prefilled from the parent document.

You can print checks. The print form is compatible with the QuickBooks "Check on Top" check format.

To enter a check

  1. In the Bank section, click Checks and then click Create.

    -OR-

    Open an Invoice payment or a Cash purchase, click Generate, and then click Check.
  2. Enter or select the following:
    • Number. The system assigns consecutive check numbers for each bank account. 
    • Pay to the order of. The company that you are paying.

      If the check will be used to register miscellaneous expense payments, leave this field blank.
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings.
    • Exchange rate (shown as currency symbols, for example, $/1€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the check date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate amounts in the line items, therefore it is recommended that you specify the date and currency before entering the line items.

      This field is only available if the Multicurrency check box is selected in Accounting settings.  
    • Date. The check date.

      This field is prefilled with the current date.
    • Amount FCY. The check amount in the transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings. 
    • Amount. The check amount in the reporting currency.
    • Bank account. The account (e.g. bank, cash, credit card, PayPal, etc.) that will receive the funds. 
    • Based on. The document from which the check is generated: an Invoice payment or a Cash purchase.
  3. In line items, select G/L accounts and amounts you would like to pay.

    Note that line items will be unavailable if the check is created based on an Invoice payment or a Cash purchase.
  4. In the Memo field, you can enter notes on the check.
  5. Click Post and close.
     

To print a check

  • Open the check and click Print check.
