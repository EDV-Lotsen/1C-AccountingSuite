The purchase invoice document records your liability to a vendor, records expenses, and updates inventory balances. It also can be used for entering beginning balances.

Purchase invoices generate general journal transactions. For detailed information about transactions that are generated when a purchase invoice is posted, see the General journal help section.

You have the option to generate a purchase invoice document from one or several Purchase order documents. In this case most of the purchase invoice fields are automatically filled from the purchase orders.

To enter a purchase invoice

  1. In the Purchasing section, click Purchase invoices and then click Create.

    -OR-

    In a Purchase order document, click Generate and then click Purchase invoice.
  2. Enter or select the following:
    • Number. By default the system assigns consecutive numbers when documents are saved, and no user input is required.

      You have the option to enter a custom number instead. Document numbers can include prefixes. For example, if you specify the number "PI-1", your next purchase invoices are automatically numbered "PI-2", "PI-3", and so on.  
    • Currency. The transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • A/P account. The accounts payable account used for G/L posting.
    • Terms. The payment term.
    • Price includes VAT. Indicates whether the prices specified in the purchase invoice include VAT.

      This check box is only available if the VAT check box is selected in Accounting settings. 
    • Date. The purchase invoice date.

      This field is prefilled with the current date.
    • Exchange rate (shown as currency symbols, for example, $/€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the purchase invoice date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate prices in the line items, therefore it is recommended that you specify the date and currency before entering the line items.

      This field is only available if the Multicurrency check box is selected in Accounting settings.  
    • Ref #. The vendor invoice number.
    • Due date. The payment due date.

      This field is calculated automatically based on the document date and the Terms field.
    • Vendor. The vendor (here you can only select companies of the vendor type).

      Once you select a vendor, a list of open and backordered purchase orders for this vendor is displayed. Select the purchase orders you want to fulfill with this purchase invoice.

      The field name can be customized (for example, to Supplier) in the Vendor name field in Accounting settings. 
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings. 
    • Location. The location of the warehouse where the purchase is delivered.

      This field is only available if the Multilocation check box is selected in Accounting settings. 
  3. On the Line items tab, you can edit the list of items, and specify their prices and quantities.

    If the purchase invoice is created based on one or more purchase orders, items are prefilled from the selected purchase orders. Item prices are prefilled with item last costs. The system automatically calculates a line total for each item.

    If the VAT check box is selected in Accounting settings, a default VAT code is prefilled for each item.

  4. On the Expenses tab, you can record purchases by entering G/L accounts and amounts.

    This option is handy if item tracking is not required, for example, for entering office supplies purchases on account.  

  5. Enter or select the following:
    • Memo. In this field you can enter notes on the purchase invoice.
    • Beginning balance. Indicates whether the purchase invoice is used for entering beginning balances.

      This field is only available if the Beg. balances in documents check box is selected in Accounting settings.
  6. Click Post and close.

    The informational (read-only) section at the bottom of the form contains the following data:
    • VAT. VAT calculated based on the VAT codes assigned to items.

      When you post the purchase invoice, a VAT liability is recorded to the General journal.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Total FCY. The total amount in the transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings. 
    • Total. The total amount in the reporting currency.
