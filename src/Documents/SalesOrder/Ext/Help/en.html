The sales order is the central document in the sales workflow. The system tracks shipped and invoiced items against sales orders.

Sales orders do not generate General journal transactions.

To enter a sales order

  1. In the Sales section, click Sales orders and then click Create.
  2. Enter or select the following:
    • Number. By default the system assigns consecutive numbers when a document is saved, and no user input is required.

      You have the option to enter a custom number instead. Document numbers can include prefixes. For example, if you specify the number "SO-1", your next sales orders are automatically numbered "SO-2", "SO-3", and so on. 
    • Currency. The transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • Promised date. Promised date for the items to be delivered or services performed. 
    • Order status. The order status (read-only) that can be one of the following:
      • New. The sales order is being created.
      • Open. The items have not been shipped or invoiced yet.
      • Backordered. The items are partially shipped and invoiced.
      • Closed. The items are shipped and invoiced.
    • Date. The sales order date.

      This field is prefilled with the current date.
    • Exchange rate (shown as currency symbols, for example, $/€). The exchange rate between the transaction currency and the reporting currency.

      The system retrieves the exchange rate from the Exchange rates list.

      If you change the sales order date, the system recalculates the exchange rate. If you change the date or currency, the system does not recalculate prices in the line items, therefore it is recommended that you specify the date and currency before entering the line items.

      This field is only available if the Multicurrency check box is selected in Accounting settings.
    • Ship to. The shipping address.

      This field is prefilled with the customer's default shipping address.
    • Confirm to. The shipment notification address/contact. 
    • Customer. The customer (here you can only select companies of the customer type).

      The field name can be customized (for example, to Client) in the Customer name field in Accounting settings. 
    • Company code. The company code (read-only).

      This field is only available if the Display company codes check box is selected in Accounting settings. 
    • Location. The location of the warehouse where the sold items are stored.

      This field is only available if the Multilocation check box is selected in Accounting settings. 
  3. On the Line items tab, select the items included in the sales order, and specify their prices and quantities.

    Item prices are automatically prefilled from the Price list. The system automatically calculates a line total for each item.

    If the VAT check box is selected in Accounting settings, a default VAT code is prefilled for each item.

    If the Sales tax (US) check box is selected in Accounting settings, the default sales tax type is prefilled for each item (by default, inventory items are taxable and noninventory items are nontaxable).

    The Backorder field is filled automatically with the quantity that is not yet shipped or invoiced. If the order has the Open status, the Backorder quantity is 0. The Shipped and Invoiced columns are filled automatically when a sales invoice based on the sales order is created and posted.
  4. Enter or select the following:
    • Memo. In this field you can enter notes on the sales order.
    • Price includes VAT. Indicates whether the prices specified in the sales order include VAT.

      This field is only available if the VAT check box is selected in Accounting settings. 
  5. Click Post and close.

    The informational (read-only) section at the bottom of the form contains the following data:
    • Sales tax. The sales tax based on the sales tax code assigned to the customer company and the sales tax types assigned to line items.
    • VAT. VAT calculated based on the VAT codes assigned to items.

      When you post the sales invoice based on this sales order, a VAT liability is recorded to the General journal.

      This field is only available if the VAT check box is selected in Accounting settings.
    • Total FCY. The total amount in the transaction currency.

      This field is only available if the Multicurrency check box is selected in Accounting settings. 
    • Total. The total amount in the reporting currency.
